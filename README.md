# TriForge

> Tri（三线）+ Forge（锻造）— 三条线共同锻造一个产品。
>
> 面向一人团队的多 Agent 协作工作体系，让独立开发者获得团队级决策质量。

---

## 概述

TriForge 是一套面向"一人团队"的 AI Agent 多角色协作框架。它将完整的产品/设计/技术团队分工抽象为 15 个 Agent 角色 + 1 套共识对冲机制，通过结构化的提示词和文件流转，让一个人借助 AI 完成从需求到上线的全流程。

### 解决的问题

- **决策质量**：一个人做产品，容易拍脑袋、漏需求、设计和技术脱节
- **信息断层**：多 Agent 是单向流水线，前一棒丢给后一棒，信息损耗逐级放大
- **冲突后置**：问题在开发过程中意外暴露，返工成本高

### 核心思想

将产品/设计/技术三线的信息不对称和目标冲突从"意外"变成"流程的一部分"——在开发的**决策阶段**（而非执行阶段）强制暴露和解决冲突。

---

## 三层协作模型

```
你（用户）—— 最终决策者
    │ 对话
    ▼
总理 Agent（战略层）
    │ 秘书 / 总项目经理 / 总参谋
    │ 告诉你下一步该找谁
    ▼
SOLO（执行层 · 传令官）
    │ 自动调度 Sub Agent，每个独立上下文
    ▼
Sub Agent（干活 → 向你汇报 → 你回总理更新进度）
```

**指南 Agent** 独立于三层模型之外，与总理 Agent 同级。不参与项目执行，只回答关于 TriForge 体系运行的问题。

### 核心规则

- Sub Agent 之间不能互调，只能由 SOLO 统一调度
- 你直接找 Sub Agent 时，它通过同步文件让总理自动对齐（不依赖对话记忆）
- 所有 Agent 产出一份确认一份，不批量输出

## 架构总览

```
主控层
├── 总理 Agent      — 秘书/总项目经理/总参谋，你的全局窗口
└── 指南 Agent      — 体系向导（平行，不参与执行）

产品线（产品经理 Agent 统筹）
├── 需求分析师      ① → 需求池
├── 竞品分析师      ② → 竞品分析
├── 用户研究员      ③ → 用户画像
├── 功能架构师      ④ → 定位 / 链路 / 逻辑
└── PRD 撰写师      ⑤ → 信息架构 / PRD

设计线（设计师 Agent 一人全包）
  需求解析 → 设计系统 → 交互稿 → 视觉稿 → 设计规范

技术线（技术经理 Agent 统筹）
├── 前端工程师      — 前端代码实现
├── 后端工程师      — 后端代码实现 + 部署指导
├── 数据库工程师    — SQL 脚本
└── 测试 Agent      — 验收报告
```

### 共识与对冲节点

三个交叉校验节点，确保所有冲突在技术执行前被识别和解决：

- **产品-设计共识**：PRD 页面结构 + 设计系统对齐
- **产品-技术对冲**：PRD + 技术方案讨论，技术栈选型在此完成
- **设计-技术共识**：设计规范 + 技术方案对齐

分歧无法解决时升级到用户仲裁。

## 快速开始

### 从 GitHub 模板创建

1. 打开本仓库页面，点击绿色的 **"Use this template"** 按钮
2. 输入你的项目名称，点击 **"Create repository from template"**
3. 将新仓库克隆到本地：

```bash
git clone git@github.com:你的用户名/你的项目名.git
cd 你的项目名
```

### 初始化后

```
你的新项目/
├── backend/                    ← 等着你放后端代码
├── frontend/                   ← 等着你放前端代码
├── database/                   ← 等着你放 SQL 脚本
├── _tri/                       ← TriForge 心智空间
│   ├── agents/                 ← 15 份 Agent 提示词（按三线分目录）
│   ├── memory/                 ← 共享上下文
│   │   ├── PROJECT_BRIEF.md    ← 请填写
│   │   └── 环境与约定说明.md   ← 请填写
│   └── output/                 ← 各线产出物目录
└── templates/                  ← 可复用模板
```

### 配置 Agent

1. 填写 `_tri/memory/PROJECT_BRIEF.md`：产品背景、定位、目标用户
2. 填写 `_tri/memory/环境与约定说明.md`：你的开发环境（JDK/IDE/数据库）
3. 在 AI 编程工具（Trae / Claude Code / Cursor）中创建各 Agent，粘贴对应提示词
4. **总理 Agent** 第一个配，启动后它会告诉你下一步该找谁

## Agent 一览

| Agent | 定位 | 配置优先级 |
|-------|------|-----------|
| 总理 Agent | 秘书/总项目经理/总参谋 | P0 最先 |
| 指南 Agent | 体系向导 | P0 可选 |
| 产品经理 Agent | 产品线负责人 | P1 |
| 需求分析师 Agent | 需求拆解 | ① |
| 竞品分析师 Agent | 竞品调研 | ② |
| 用户研究员 Agent | 用户画像 | ③ |
| 功能架构师 Agent | 功能设计 | ④ |
| PRD 撰写师 Agent | PRD 文档 | ⑤ |
| 设计师 Agent | 设计线全流程 | P2 |
| 技术经理 Agent | 技术线负责人 | P2 |
| 前端工程师 Agent | 代码实现 | 按需 |
| 后端工程师 Agent | 代码实现 + 部署 | 按需 |
| 数据库工程师 Agent | SQL 设计 | 按需 |
| 测试 Agent | 质量保障 | 开发完成后 |

## 你必须决策的 6 个节点

| # | 决策 | 时机 |
|---|------|------|
| 1 | PRD 定稿 | 产品线产出后 |
| 2 | 设计定稿 | 设计线产出后 |
| 3 | 技术栈选择 | 技术经理推荐方案后 |
| 4 | 变更请求裁定 | 项目进行中随时 |
| 5 | 共识/对冲仲裁 | Agent 分歧无法解决 |
| 6 | 上线/迭代 | 验收完成后 |

## 环境要求

无。TriForge 的核心资产是提示词和目录结构，不绑定任何 AI 编程工具、编程语言或 IDE。你可以在 Trae、Claude Code、Cursor 中使用，也可以跨工具切换。
