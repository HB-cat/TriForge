# TriForge 使用说明书

> 不用懂编程，按步骤操作就行。

---

## 这是什么

TriForge 是一套帮你管理产品开发流程的工具。它把"产品经理、设计师、技术经理、工程师"这些角色的工作方法打包成了 15 份操作指南（提示词），配合一套文件目录结构，让一个人也能像团队一样工作。

**一句话：它教你一个人怎么有章法地做产品。**

---

## 快速上手（10 分钟）

### 第一步：获取 TriForge

打开 https://github.com/HB-cat/TriForge

点击绿色的 **"Use this template"** 按钮 → 输入你的项目名称 → 创建。

然后在电脑上打开终端（命令行），输入：

```bash
git clone ssh://git@ssh.github.com:443/你的用户名/你的项目名.git
cd 你的项目名
```

如果你不会用终端，也可以直接点 **"Code" → "Download ZIP"**，下载后解压到你的项目文件夹。

### 第二步：填写项目信息

打开 `_tri/memory/PROJECT_BRIEF.md`，填写：
- 你这个产品是做什么的
- 给谁用的
- 想解决什么问题

打开 `_tri/memory/环境与约定说明.md`，填写：
- 你电脑上装了哪些开发工具（JDK、IDE 等版本）
- 数据库配置
- 项目目录位置

### 第三步：在 AI 工具中创建智能体

- 如果你用 **Trae**：在 Trae 中创建智能体，复制 `_tri/agents/` 下对应的提示词进去
- 如果你用 **Claude Code**、**Cursor** 等其他 AI 工具：同样操作，复制提示词内容创建自定义智能体

**先从总理 Agent 开始**，它是你的总入口。

### 第四步：开始工作

跟总理 Agent 对话，告诉它你要做什么产品。它会告诉你下一步该找谁。

## 文件结构说明（简单版）

你下载后会看到这些文件和文件夹：

```
backend/              ← 放后端代码的地方（将来放项目代码）
frontend/             ← 放前端代码的地方（将来放项目代码）
database/             ← 放 SQL 脚本的地方（将来放项目代码）

_tri/                 ← 核心工作区（所有智能体的工作空间）
  ├── agents/         ← 15 份智能体操作指南（提示词）
  ├── memory/         ← 项目信息（你来填写）
  │   ├── PROJECT_BRIEF.md    ← 产品简介
  │   ├── 环境与约定说明.md    ← 你的电脑配置
  │   ├── PROJECT_MEMORY.md   ← 项目进度（智能体自动更新）
  │   ├── TASK_QUEUE.md       ← 任务列表（智能体自动更新）
  │   └── TECH_DECISIONS.md   ← 技术决策记录（智能体自动更新）
  ├── output/         ← 智能体写好的文档放在这
  │   ├── product/    ← 产品文档（需求池、竞品分析等）
  │   ├── design/     ← 设计文档（设计稿、设计规范等）
  │   ├── tech/       ← 技术文档（技术方案、排期等）
  │   └── test/       ← 测试文档（测试用例、验收报告）

templates/            ← 可以重复使用的模板
```

## 智能体（Agent）清单

你一共会创建 15 个智能体，每个扮演一个角色：

### 两个"总指挥"（先配置）

| 智能体 | 作用 | 什么时候用 |
|--------|------|-----------|
| **总理 Agent** | 你的秘书+项目经理。跟它对话了解全局进度 | 每次使用前先找它 |
| **指南 Agent** | 使用说明书（就是你现在看的这个），不懂流程问它 | 不确定怎么做时找它 |

### 产品线（6 个，依次使用）

按顺序找它们，每完成一个就回来跟总理汇报：

| 智能体 | 做什么 | 产出物 |
|--------|-------|--------|
| **产品经理** | 产品线的负责人，调度下面 5 个智能体 | 最终汇总 PRD |
| **需求分析师** | 把你的想法整理成结构化的需求清单 | 需求池 |
| **竞品分析师** | 搜索同行产品怎么做，分析优缺点 | 竞品分析报告 |
| **用户研究员** | 描述你的目标用户长什么样、在什么场景下使用 | 用户画像 |
| **功能架构师** | 画流程图、定功能规则、设计产品逻辑 | 三份设计文档 |
| **PRD 撰写师** | 把上面所有的整合成一份完整的产品需求文档（PRD） | 产品需求文档 |

### 设计线（1 个）

| 智能体 | 做什么 |
|--------|-------|
| **设计师** | 从分析需求到出设计图、定设计规范、到最后验收，一人全包 |

### 技术线（5 个，由技术经理统筹）

| 智能体 | 做什么 |
|--------|-------|
| **技术经理** | 技术线负责人，评估能不能做、怎么做、排时间 |
| **前端工程师** | 写网页/App 界面代码 |
| **后端工程师** | 写服务器代码、提供接口、部署上线 |
| **数据库工程师** | 设计数据库表结构 |
| **测试** | 测试功能有没有 Bug，出验收报告 |

## 运行流程（六步走）

### 第一步：启动

跟总理 Agent 说你要做什么产品。总理会拆解任务，告诉你先找谁。

### 第二步：产品定义

按顺序找产品线的 5 个智能体——需求分析师 → 竞品分析师 → 用户研究员 → 功能架构师 → PRD 撰写师。

**每完成一步，回来跟总理确认，再走下一步。**

全部完成后，你来决定 PRD 是否通过。

### 第三步：设计与技术并行

PRD 通过后，设计师和技术经理同时开始工作。

**三个检查节点：**
- 产品经理 vs 设计师：PRD 和设计图是否对齐
- 产品经理 vs 技术经理：PRD 和技术方案是否冲突（这里要你来定技术栈）
- 设计师 vs 技术经理：设计稿能不能实现

### 第四步：开发

前端、后端、数据库工程师开始写代码。

开发过程中如果改需求，先回总理，你来决定。

### 第五步：验收

测试智能体出验收报告 → 你决定上线还是继续改。

**复盘：** 总理会问你三个问题——这次做得好的是什么？踩了什么坑？下次怎么改进？

### 第六步：沉淀

把做得好的文档变成模板，存到 `templates/` 里，以后的项目可以用。

---

## 你需要做的 6 个决策

在整个过程中，有 6 件事需要你亲自拍板：

| # | 你决定什么 | 什么时候决定 |
|---|-----------|------------|
| 1 | PRD 能不能通过 | 产品线工作完成后 |
| 2 | 设计稿能不能通过 | 设计线工作完成后 |
| 3 | 用什么技术（前后端用什么语言/框架） | 技术经理推荐方案后 |
| 4 | 改不改需求、加不加功能 | 项目进行中随时 |
| 5 | 产品经理和技术经理吵起来时听谁的 | 分歧无法解决时 |
| 6 | 上线还是继续改 | 验收完成后 |

## 常见问题

### 我能直接找某个智能体，不通过总理吗？

可以。智能体完成任务后会自动同步进度，下次你找总理时它会知道最新情况。

### 项目做了一半停了，回来怎么继续？

直接找总理，它会读取任务记录，告诉你做到哪了。

### 我不小心跳过了一个步骤怎么办？

智能体开始工作前会检查上游文件是否完整。如果发现还没做完，它会提示你"建议先完成 XX 步骤"。你坚持要做，它也会听你的。

### 智能体之间会互相干扰吗？

不会。每个智能体有各自的工作目录。后端工程师不会去改前端的代码，产品经理不会去改数据库的脚本。

### 一定要用 Trae 吗？

不一定。这套东西的核心是提示词（文本文件），你可以用在任何支持自定义智能体的 AI 工具里。

### 我不会用终端（命令行），怎么办？

在 GitHub 上点 "Code" → "Download ZIP"，下载解压就能用。只是后续更新需要手动重新下载。
