## 二、各层级 Agent 职责明细

### 层级一：主控层（决策与调度中心）

| Agent 名称 | 角色定位 | 核心职责 | 配置优先级 |
|-----------|---------|---------|-----------|
| **总理 Agent** | 项目总控 / 调度中心 | 全局任务拆解、三线并行调度、跨线冲突识别、关键节点质量检查 | **P0 - 最先配置** |


### 层级二：主控层扩展

| Agent 名称 | 角色定位 | 核心职责 | 配置优先级 |
|-----------|---------|---------|-----------|
| **指南 Agent** | 体系向导（平行） | 解释三层模型、角色职责、规则说明、文件导航，不参与项目执行 | **P0 - 可选配置** |

### 层级三：三线协作层（负责人层级）

| Agent 名称 | 角色定位 | 核心职责 | 配置优先级 |
|-----------|---------|---------|-----------|
| **产品经理 Agent** | 产品线负责人 | 承接总理 Agent 的产品任务，调度产品线子团队，汇总产出，把控 PRD 质量 | **P1 - 总理之后配置** |
| **设计师 Agent** | 设计线负责人 | 从需求解析到设计验收的完整设计工作流统筹，确保设计可被技术精确还原 | **P2 - PRD 产出后配置** |
| **技术经理 Agent** | 技术线负责人 | 从技术评审到测试修复的完整技术工作流统筹，调度下属工程师，把控技术方案质量 | **P2 - PRD 产出后配置** |


### 层级四：产品线子团队（产品经理 Agent 下属）

| Agent 名称 | 角色定位 | 职责 | 所属节点 | 配置顺序 |
|-----------|---------|------|---------|----------|
| **需求分析师 Agent** | 需求拆解专员 | 将模糊的用户需求拆解为结构化的需求条目，标注优先级和依赖关系 | 01 需求收集 | ① |
| **竞品分析师 Agent** | 市场调研专员 | 收集并分析竞品信息，形成对比报告，提炼可借鉴功能点和差异化机会 | 02 竞品分析 | ② |
| **用户研究员 Agent** | 用户研究专员 | 构建目标用户画像和场景故事，确保产品决策以用户为中心 | 03 用户画像 | ③ |
| **功能架构师 Agent** | 功能规则设计师 | 串联产品定位、核心链路、功能逻辑，输出业务流程图、规则定义和状态机 | 04 产品定位<br>05 核心链路<br>06 功能逻辑 | ④ |
| **PRD 撰写师 Agent** | PRD 文档工程师 | 整合前述所有产出，撰写完整 PRD 文档，含页面结构、功能详情、验收标准 | 07 信息架构<br>08 PRD 撰写 | ⑤ |


### 层级五：技术线子团队（技术经理 Agent 下属）

| Agent 名称 | 角色定位 | 职责 | 配置顺序 |
|-----------|---------|------|----------|
| **前端工程师 Agent** | 前端实现 | 根据设计稿和技术方案实现前端页面组件，还原设计样式 | 按需配置 |
| **后端工程师 Agent** | 后端实现 | 根据 PRD 和技术方案实现后端 API 和业务逻辑，部署指导 | 按需配置 |
| **数据库工程师 Agent** | 数据库设计 | 根据技术方案设计数据库表结构和 SQL 脚本 | 按需配置 |
| **测试 Agent** | 质量保障 | 根据 PRD 验收标准编写测试用例，执行测试并输出验收报告，归技术经理管辖 | 开发完成后配置 |


### 层级六：资产层

| 机制名称 | 角色定位 | 职责 | 触发时机 |
|---------|---------|------|----------|
| **模板沉淀机制** | 资产管理者 | 从已完成项目中抽取可复用模板，沉淀到模板库，供后续项目复用 | **项目验收完成后自动触发** |


## 三、各 Agent 输入输出对照表

| 竞品分析师 Agent | 竞品信息、行业数据 | `02_竞品分析.md` |
| 用户研究员 Agent | 用户信息、场景描述 | `03_用户画像.md` |
| 功能架构师 Agent | 产品定位文档、用户场景 | `04_产品定位.md` → `05_核心链路.md` → `06_功能逻辑.md` |
| PRD 撰写师 Agent | 前三阶段全部产出 | `07_信息架构.md` → `08_PRD.md` |
| 设计师 Agent | `08_PRD.md` | `01_设计系统.md` → `02_交互稿.md` → `03_视觉稿_标注.md` → `04_设计规范.json` |
| 技术经理 Agent | `08_PRD.md` + `04_设计规范.json` | `01_技术评审.md` → `02_技术方案.md` → `03_排期计划.md` |
| 前端工程师 Agent | `02_技术方案.md` + `03_视觉稿_标注.md` + `04_设计规范.json` | 前端代码（`.vue` / `.js` / `.css`） |
| 后端工程师 Agent | `02_技术方案.md` + `08_PRD.md`（业务规则部分） | 后端代码（Controller / Service / DTO / Mapper） |
| 数据库工程师 Agent | `02_技术方案.md`（数据模型部分） | SQL 脚本（CREATE TABLE / INDEX / MIGRATION） |
| 测试 Agent | `08_PRD.md`（验收标准部分） | `01_测试用例.md` → `02_验收报告.md` |


## 四、Agent 配置顺序总览

```
【主控层】
  Step 1 → 总理 Agent（P0）+ 指南 Agent（P0，可选）

【产品线】
  Step 2 → 产品经理 Agent（P1）
  Step 2a → 需求分析师 Agent
  Step 2b → 竞品分析师 Agent
  Step 2c → 用户研究员 Agent
  Step 2d → 功能架构师 Agent
  Step 2e → PRD 撰写师 Agent

【设计线 + 技术线】（PRD 产出后并行）
  Step 3a → 设计师 Agent（P2）
  Step 3b → 技术经理 Agent（P2）

【技术执行】（按需）
  Step 4a → 前端工程师 Agent
  Step 4b → 后端工程师 Agent
  Step 4c → 数据库工程师 Agent
  Step 4d → 测试 Agent（归技术经理管辖）

【资产沉淀】
  Step 5 → 模板沉淀机制（项目结束后自动触发）
```

## 六、各 Agent 提示词核心内容要求

| Agent | 提示词必须包含的核心内容 |
|-------|-------------------------|
| **总理 Agent** | 任务拆解逻辑、三线调度规则、冲突识别触发条件、质量检查标准、临时任务判断、项目复盘 |
| **指南 Agent** | 体系解释、角色查询、规则说明、文件导航 |
| **产品经理 Agent** | 产品工作流统筹规则、子 Agent 调度逻辑、PRD 质量把控标准 |
| **需求分析师 Agent** | 需求结构化方法、优先级标注规则（MoSCoW 或类似框架） |
| **竞品分析师 Agent** | 竞品分析方法论、对比维度框架（功能/体验/价格/策略） |
| **用户研究员 Agent** | 用户画像构建方法、场景故事编写规范 |
| **功能架构师 Agent** | 业务流程图规范、状态机定义方法、规则描述标准 |
| **PRD 撰写师 Agent** | PRD 章节结构（背景/目标/功能详情/验收标准）、格式规范 |
| **设计师 Agent** | Design System 构建规范、Design Tokens 定义标准、设计验收标准 |
| **技术经理 Agent** | 技术评审规则、可行性评估方法、架构设计原则、任务拆解逻辑 |
| **前端工程师 Agent** | 组件规范、样式还原规则、Token 引用规范 |
| **后端工程师 Agent** | API 设计原则（RESTful）、分层架构规范（Controller/Service/Mapper） |
| **数据库工程师 Agent** | 表设计规范（范式/命名/字段类型）、索引设计原则 |
| **测试 Agent** | 测试用例设计方法（等价类/边界值/场景法）、缺陷报告规范 |

|-------|-------------|----------|

| 指南 Agent | ⬜ 可选 | 需要体系解释时启用 |
| 产品经理 Agent | ✅ 必须 | 总理 Agent 启用后启用 |
| 需求分析师 Agent | ✅ 必须 | 产品经理 Agent 启用后启用 |
| 竞品分析师 Agent | ✅ 必须 | 需求收集完成后启用 |
| 用户研究员 Agent | ✅ 必须 | 竞品分析完成后启用 |
| 功能架构师 Agent | ✅ 必须 | 用户画像完成后启用 |
| PRD 撰写师 Agent | ✅ 必须 | 功能架构师产出后启用 |
| 设计师 Agent | ✅ 强烈建议 | PRD 产出后启用 |
| 技术经理 Agent | ✅ 必须 | PRD 产出后启用 |
| 前端工程师 Agent | ⚠️ 按需 | 需要前端实现时启用 |
| 后端工程师 Agent | ⚠️ 按需 | 需要后端实现时启用 |
| 数据库工程师 Agent | ⚠️ 按需 | 需要数据库设计时启用 |
| 测试 Agent | ✅ 建议 | 开发完成后启用，归技术经理管辖 |
| 模板沉淀机制 | ✅ 建议 | 项目验收完成后触发 |
